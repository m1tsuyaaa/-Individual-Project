# Почтовый клиент (Email Client)

**Индивидуальный проект** — паттерн проектирования **Singleton** (Одиночка).

**Автор:** Катков Степан Сергеевич ПИ-251

---

## Постановка задачи

Необходимо разработать почтовый клиент, который позволяет:

- отправлять электронные письма;
- получать новые письма (симуляция);
- просматривать входящие и отправленные письма;
- отмечать письма как прочитанные;
- просматривать статистику почтового ящика.

**Ключевое требование:** гарантировать, что в приложении существует только один экземпляр почтового клиента.

Реализация: консольное приложение на **C# (.NET Framework 4.7.2)**.

---

## Реальный сценарий применения

Пользователь ежедневно работает с электронной почтой: отправляет письма коллегам, получает уведомления, отслеживает важные сообщения. Создание нескольких экземпляров почтового клиента может привести к:

- конфликту подключений к почтовому серверу;
- несогласованности состояния почтового ящика;
- избыточному потреблению ресурсов.

**Решение:**

1. При первом запуске создаётся единственный экземпляр почтового клиента.
2. Все последующие обращения возвращают тот же самый объект.
3. Это гарантирует целостность данных и экономит ресурсы.

Такой подход особенно актуален для мобильных и десктопных приложений, где важно контролировать количество подключений и состояние почтового ящика.

## Паттерн Singleton (Одиночка)

### Зачем нужен

Создание нескольких экземпляров почтового клиента может привести к проблемам: разные объекты будут иметь свои копии входящих писем, что нарушит целостность данных.

**Singleton** гарантирует, что класс имеет только один экземпляр, и предоставляет глобальную точку доступа к нему.

### Какую проблему решает в проекте

| Проблема | Решение через Singleton |
|----------|------------------------|
| Несколько подключений к почтовому серверу | Только один клиент управляет соединением |
| Несогласованное состояние почтового ящика | Все операции через один экземпляр → консистентность |
| Конфликты при одновременной работе | Потокобезопасная реализация с блокировкой |
| Избыточное потребление ресурсов | Экземпляр создаётся один раз |

### Ключевые элементы

- `EmailClient` — Singleton класс с приватным конструктором.
- `GetInstance()` — статический метод для получения единственного экземпляра.
- `threadLock` — объект для потокобезопасной инициализации (double-check locking).

### Реализация в коде

```csharp
public sealed class EmailClient {
    private static EmailClient singleInstance;
    private static readonly object threadLock = new object();

    private EmailClient() { }

    public static EmailClient GetInstance() {
        if (singleInstance == null) {
            lock (threadLock) {
                if (singleInstance == null) {
                    singleInstance = new EmailClient();
                }
            }
        }
        return singleInstance;
    }
}
```

### Пункт 1 — отправка письма

Ввести email получателя.
Ввести тему письма.
Ввести содержание письма.
Письмо сохраняется в отправленных.

### Пункт 2 — получение новых писем

Симулирует получение 1-3 новых случайных писем. Письма добавляются во входящие.

### Пункт 3 — просмотр входящих

Отображает все письма во входящих с указанием статуса [NEW] или [READ].

### Пункт 4 — просмотр отправленных

Отображает все отправленные письма.

### Пункт 5 — отметка о прочтении

Выбрать номер письма из списка входящих.
Письмо отмечается как прочитанное.

### Пункт 6 — статистика
Показывает:
общее количество писем;
количество входящих и непрочитанных;
количество отправленных.

### Пункт 0 — выход
Завершение работы приложения.
